# Engagement Kickoff Instructions

<workflow>
  <step n="1" goal="Confirm context">
    <action>Verify inputs: project name, client profile, engagement type, timeline, success metrics.</action>
    <action>Load any provided briefs or contracts if referenced.</action>
  </step>
  <step n="2" goal="Stakeholder alignment">
    <action>List core stakeholders and roles (client, farmers, partners, tech team).</action>
    <action>Capture communication cadences and languages.</action>
  </step>
  <step n="3" goal="Define mission & outcomes">
    <action>Clarify problem statement, desired outcomes, SDG alignment, and product scope.</action>
    <action>Identify constraints (budget, data access, tech stack, regulations).</action>
  </step>
  <step n="4" goal="Map success metrics">
    <action>Define primary metrics (impact, adoption, revenue) and supporting indicators.</action>
    <action>Set checkpoints for validation (design reviews, field tests, pilot launch).</action>
  </step>
  <step n="5" goal="Plan next steps">
    <action>Outline immediate next actions for each module agent.</action>
    <action>Highlight key risks or open questions.</action>
  </step>
  <step n="6" goal="Generate kickoff canvas">
    <action>Populate template with findings and save to {output_folder}.</action>
  </step>
</workflow>
